# Lesson 10: Loop Capstone + AI Safety

## Duration
- 60 minutes total
- 40 minutes coding

## Learning Goals
- Combine `for` and `while` in one program
- Use loops for simulation and validation
- Use Gemini responsibly with auditable prompts
- Demonstrate understanding through explanation and tests

## Scenario Brief (Explicit)
Theme: Multiverse portal rotation control.
Students simulate 6 portal cycles (`for`) and validate command input (`while`).

Required elements:
- `for` loop for fixed cycles
- `while` loop for command validation (`open`/`hold`)
- Running totals for open vs hold commands
- Final mission summary

## End-of-Lesson Deliverable (Measurable)
1. Program runs 6 cycles
2. Invalid command is re-prompted until valid
3. Final totals printed for both command types
4. Student submits 2 good prompts + 1 weak prompt rewrite
5. Student submits 3 test cases with expected vs actual output

## Good vs Weak Gemini Prompts
Weak:
- "Build the whole portal system for me"
Good:
- "I already wrote my loop. Help only with validating `open`/`hold` input using `while`, and explain why it avoids bad commands."

## Anti-Cheat Check
Teacher asks each student:
1. Why does your `while` loop stop?
2. What changes each `for` cycle?
3. Which prompt improved your code quality and why?

## Schedule
- 0-10 min: capstone requirements + AI evidence rules
- 10-20 min: guided tutorial
- 20-55 min: independent build
- 55-60 min: viva check
