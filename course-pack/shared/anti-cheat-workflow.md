# Anti-Cheat Workflow

Use this workflow every lesson.

## Required Submission
1. Final code text
2. Screenshot of one failed run
3. Screenshot of corrected run
4. Three-sentence debugging note

## Teacher Controls
1. Assign unique numeric seeds per student or pair
2. Change one scenario variable per table group
3. Randomly viva-check one branch condition per student

## Why This Works
- Objective questions verify concept recall
- Reflection requires personal process evidence
- Personalized inputs make direct copy-paste less useful
