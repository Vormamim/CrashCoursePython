# Lesson 1: Mission Control (Arena Profile)

## Duration
- 60 minutes total
- 40 minutes coding

## Learning Goals
- Use print statements to communicate output
- Create and update variables
- Identify int, float, str, and bool values
- Build simple expressions

## Scenario Brief (Explicit)
You are coding an arena control screen for one tribute before a match starts.
Students must track one character's profile and show a readable status panel.

Required scenario data:
- Tribute name (text)
- District number (whole number)
- Starting score (decimal)
- Active status (True/False)

Required scenario event:
- "Supply drop bonus" increases score once during runtime.

## End-of-Lesson Deliverable (Measurable)
Each student submits a program that:
1. Prints a mission title
2. Defines all 4 required variables with correct types
3. Reassigns score after one event
4. Prints a 4-line final report with labels
5. Runs without errors in front of teacher

## Theory (Clear and Relatable)
- `Variables` are labeled boxes that store information your program needs.
- `Types` describe what kind of information is in each box: `int`, `float`, `str`, `bool`.
- `Reassignment` means updating a variable when the story changes.

### Relatable Examples
- Arena health points: start with `health = 10`, then update after damage.
- Movie fan rating: `rating = 8.5` is a `float` because it includes decimals.
- Team tag: `team_name = "Raven"` is a `str` because text uses quotes.
- Mission state: `is_active = True` is a `bool` because it is either true or false.

## Schedule
- 0-8 min: Hook (arena briefing console)
- 8-18 min: Teach variables, types, reassignment
- 18-58 min: Coding mission
- 58-60 min: Exit check

## Coding Mission (40 min)
Build a "Tribute Profile Console" that:
1. Stores name, district, score, and active status
2. Updates score after an event
3. Prints a clear status report

## Objective Questions
1. What type is 9.5?
2. After lives = 3 then lives = lives - 1, what is lives?
3. True or false: variable names can start with a number.

## Reflective Prompts
1. Which variable name did you improve and why?
2. What bug did you hit first, and how did you fix it?
