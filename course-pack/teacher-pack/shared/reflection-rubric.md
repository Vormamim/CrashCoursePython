# Reflection Rubric (4 Points)

Use this rubric for each lesson reflection response.

## 4 - Strong Evidence
- Names a real bug or decision from the student's own code
- Explains cause clearly
- Describes fix or alternative with clear reasoning
- Uses correct Python vocabulary

## 3 - Good Evidence
- Mentions a real issue or decision
- Explanation is mostly correct
- Fix or next step is present but not fully explained

## 2 - Partial Evidence
- Generic response with limited connection to code
- Cause and fix are unclear or mixed

## 1 - Weak Evidence
- Vague statement with no concrete coding evidence
- Reads as copied or off-topic
