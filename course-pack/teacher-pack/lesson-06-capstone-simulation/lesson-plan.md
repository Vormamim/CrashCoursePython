# Lesson 6: Capstone Simulation (Franchise Finale)

## Duration
- 60 minutes total
- 40 minutes coding

## Learning Goals
- Plan and build a complete mini-simulator
- Use clear variable names and comments
- Test against a short test-case table
- Explain design choices with evidence

## Scenario Brief (Explicit)
You are lead developer for a YA-movie simulation challenge day.
Each student chooses one fixed scenario and builds a complete decision program.

Choose one scenario:
1. Arena survival checkpoint
2. Maze gate terminal
3. House ranking board
4. Portal clearance gate

Required capstone inputs:
- At least 3 user inputs

Required capstone processing:
- At least 1 computed value from arithmetic
- At least 3 decision outcomes in `if/elif/else`

Required capstone data structure:
- At least one meaningful list or tuple

Required capstone output:
- Final report with key inputs, computed value, and outcome

## End-of-Lesson Deliverable (Measurable)
Each student submits a program that:
1. Meets all four required features from the worksheet
2. Includes a completed 3-case test table (expected vs actual)
3. Produces one valid final outcome per test
4. Includes short reflection evidence on one bug and one design choice
5. Runs in class demo without forbidden topics

## Theory (Clear and Relatable)
- `Planning before coding` reduces errors and saves time.
- A `test table` predicts what output should happen for chosen inputs.
- `Design choices` are justified decisions about names, rules, and thresholds.

### Relatable Examples
- Planning a short film scene: storyboard first, then shoot.
- Game level testing: try easy, medium, and extreme inputs before launch.
- Team challenge: explain why your scoring rule is fair and easy to understand.

## Schedule
- 0-8 min: Hook and capstone options
- 8-18 min: Planning and test-case setup
- 18-58 min: Build and test
- 58-60 min: Exit check

## Coding Mission (40 min)
Students choose one scenario:
1. Arena survival checkpoint
2. Maze escape terminal
3. House ranking board
4. Portal clearance gate

Program must include:
- Variables and reassignment
- input() and conversion
- if/elif/else decisions
- list or tuple usage

## Objective Questions
1. List the four required features.
2. Give one invalid input your program should handle.
3. Which part of code decides the final outcome?

## Reflective Prompts
1. What design choice in your code is uniquely yours?
2. If you had 20 more minutes, what would you improve?
