# Teacher Key: Lesson 6

## Expected Concepts
- Integration of all covered topics under constraints
- Evidence of planning before coding
- Evidence of testing and revision

## Objective Guidance
1. Required features should all be listed accurately.
2. Student should identify if/elif/else block as outcome logic.
3. Writing expected outputs first improves test quality and reasoning.

## Strong Reflection Indicators
- Specific design rationale
- Specific failed test and code fix

## Common Errors
- Overly complex design without test plan
- Missing conversion for one input
- Branches that do not cover all realistic cases
